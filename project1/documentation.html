Overall Process:

  • September 16th-18th, 2017:

    In this period I created my game proposal content and images and stored them in a markdown file.

  • September 25th, 2017:

    During this time I sketched out the layout of my website on some paper and developed a large amount of the HTML code for all pages of my website.

  • September 26th, 2017:

    I worked heavily on refining the HTML and added basic CSS. There was still no images at this point and only placeholder colors. However, most of all of the written content (excluding documentation) was located in its appropriate sections. By this time there was a navigation system to travel between pages.

  • September 27th, 2017:

    I continued working on the HTML and CSS code, refining proximity and alignment. I finished an in-page navigation system to immediately link to specific headers. Additionally there was a critique session between classmates that provided me with good feedback regarding the pros and cons of the site. The pros included good alignment, proximity, navigation and content depth while the cons centered on poor contrast, bad colors and a lack of responsive design.

  • September 28th, 2017:

    I changed colors of text and blocks to be more appealing and to fix contrast. Additionally, I chose better images (with a nature theme) and darkened them to make sure that they wouldn't detract from the written content, fixed some issues that were caused by invalid HTML and CSS, and started working with media queries.

  • September 29th, 2017

    I finished up the remainder of the project (I finalized media queries, finished implementing navigation for mobile, fixed the image resolutions, finished adding documentation, spell-checked content, finished commenting and fixed some various bugs).

Requirements:

Pages:

  1. Requirement #1: Index.html

    My index page contains a brief overview of my project, contains an image

  2. Requirement #2: Proposal.html

    I went over the proposal example document and covered all of the required fields in what I felt to be sufficient detail. I utilized many of the semantic tags mentioned in the requirements

  3. Requirement #3: Documentation.html

    I documented my overall process and resources used. I cited all of the sources of media that I used (and didn't cite any tutorials because I didn't use any apart from the W3C. This page describes how I met all of the project requirements.

  4. Requirement #4: project.html

    I have the project placeholder content (the game viewport size and controls) and similar repetition to the other pages. The main difference is is that there are vertical banners on this page, which was done intentionally to minimize distraction.

Content:

  • The cite utilizes the header, section, article, nav and footer tags apropriately.
  • There is a consistent structure throughout all of the four pages, with elements being placed in the same positions and the same colors.
  • There is a descriptive title on every page.
  • All the style rules that I know of are located on the external style sheet.
  • Many different IDs and classes were used.
  • The website uses a:link, a:hover, a:active and a:visited.
  • No inline styles were used.

Design:

  • I have definitely produced college level work.
  • My website doesn't represent any templates or in class exercises. I utilized adobe color wheel to choose an apropriate color scheme. The alignment and proximity of the elements of my page were the product of immense effort and care. A large amount of time went into making sure that the fonts of the page were consistent.
  • I have given my best effort in attempting to produce a website whose standards match the C.R.A.P. principles: The content has been well thought out and scrutinized. The presence of repetition is obvious on every page. All of the content is alligned vertically at the middle of the page. Lastly, the proximity of most all of the content and containers of the page has been thoughtfully chosen.
  • Inter-page navigation is consistent between all pages of the site while the only page that doesn't have an index is the Game Page. This is because the banners were removed to prevent distraction. Additionally, the page isn't long enough to where having an index would really matter. With this being said, the viewer can always easily determine where they are and where they want to go.
  • Text size is sufficient and easy to read.
  • The media queries that I utilized are able to support a wide variety of screen widths, including those 480px and below. The queries resize the elements of the screen (and remove them) to facilitate the user's interaction with the website.

Other:

  • I didn't use a CSS or HTML template.
  • No tables were used within the final product of this assignment. At one point they were used for navigation, although they have since been removed.
  • No plugin content was used.
  • The site is accessible without graphics enabled.
  • None of the images are via code and all images are in their native resolution.
  • The images used on this website are saved in web-friendly formats (JPEGs and PNGs)
